# Problem Description
## Background

Law enforcement agencies spend a significant amount of time preparing repetitive and detail-heavy documentation throughout the lifecycle of a criminal case. From the moment a victim applies at the police station to the eventual arrest and remand of the accused, various documents such as charge sheets, remand requests, seizure receipts, medical letters, court custody letters, and face identification forms must be created. Much of the content in these documents overlaps, yet officers must manually re-enter and verify the same information across different forms, increasing the risk of errors, omissions, and inefficiencies. Additionally, officers often struggle to immediately determine applicable legal provisions or landmark judgments based on the narrative of the incident, slowing investigation and prosecution.

---
## Problem Statement
Develop a smart AI-driven platform, CrimeGPT, that automates and accelerates the creation of crime-related documentation by eliminating duplication of information and intelligently extracting and mapping legal sections and relevant case laws based on incident narratives. The platform should assist officers from the first victim report to arrest, helping generate all required documents dynamically and maintain a case diary over time.

---
## Key Objectives
- [x] **Reduce redundant data entry** across legal documents using a unified case pool.
- [x] **Dynamically generate all required documents** for a criminal case using shared case data.
- [x] **Maintain a digital Case Diary** from FIR to arrest.
- [x] **Recommend relevant legal sections and case laws/judgments** based on crime descriptions (BNS/BNSS/BSA).
- [x] **Ensure document format compliance** with Indian criminal procedure standards.

---
## Functional Requirements

### Unified Case Data Pool:
- [x] Single entry of names, addresses, sections, items seized, statements, etc., used across documents.
- [x] Editable and traceable entries.

### Document Generation Engine:
Automatically generate the following documents:
- [x] Purvani Chargesheet
- [x] Medical Treatment Letter
- [x] Remand Request Letter (Police Custody)
- [x] Seizure Receipt
- [x] Court Custody Letter
- [x] Accused Panchanama
- [x] Accused Face Identification Form

### Case Diary Automation:
- [x] Maintain a timeline-based case diary from initial complaint to arrest.
- [x] Log investigative steps, witness interaction, evidence seizure, etc.

### Legal Section Intelligence:
NLP-based module to analyse the case summary and suggest:
- [x] Relevant sections under BNS, BNSS, BSA
- [x] Applicable landmark judgments
- [x] Cross-referenced IPC/CrPC/Evidence Act provisions where needed.

### Multilingual Interface:
- [x] Input/output support in Gujarati, Hindi, and English.

### Search & Audit:
- [x] Officers can retrieve old entries/documents using keywords or case numbers.
- [x] Version history and audit trail for all document edits.

## Suggested Tools/Technologies Used
- **Frontend:** React.js + Vite + Framer Motion + Custom CSS/Tailwind
- **Backend:** FastAPI (Python)
- **Database:** PostgreSQL (Relational) + Redis (Cache/Session) + ChromaDB (Vector Store RAG)
- **NLP & AI:** Groq (Llama 3.1 8B Instant) / Ollama fallback + Legal RAG pipeline
- **Localization:** Multi-lingual NLP & Translation pipeline (EN / HI / GU)
- **Legal Dataset:** BNS, BNSS, BSA datasets & Indian Kanoon legal texts
